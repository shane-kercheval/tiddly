## Overview

Build a Go CLI tool (`tiddly`) that eliminates manual setup friction for Tiddly integrations. Users currently have to: create PATs in the web UI, find config file paths per OS/tool, manually edit JSON/TOML, restart apps, and run curl commands. The CLI automates all of this. Scope is focused on setup automation (auth, MCP install, skills sync), data export, and token management — not content CRUD (available via web UI and MCP servers).

### Key Design Decisions

**Go over TypeScript/Python:** Single binary with zero runtime dependencies. Claude Desktop MCP setup already requires `npx mcp-remote` -- adding another Node dependency is circular. Python's `pip install` has venv confusion. Go binary has <10ms startup, trivial cross-compilation, and the best CLI/TUI ecosystem (Cobra, Charm).

**OAuth Device Flow + PAT dual auth:** Users choose: `tiddly login` opens browser via Auth0 device flow (easiest), or `tiddly login --token bm_xxx` for direct PAT login. OAuth is required for `tiddly mcp install` to auto-create PATs (since `/tokens/` endpoints are Auth0-only, 403 for PATs). PAT login works for all other commands.

**Monorepo placement:** `cli/` directory at repo root (like `chrome-extension/`). The CLI is a pure HTTP client -- no shared code with the Python backend.

### Go Libraries

| Purpose | Library | Why |
|---------|---------|-----|
| CLI framework | `github.com/spf13/cobra` | Industry standard (used by kubectl, gh, hugo). Subcommands, auto-help, completions |
| Config files | `github.com/spf13/viper` | YAML/TOML/JSON config, env var binding, works with Cobra |
| HTTP client | `net/http` (stdlib) | No dependency needed for REST calls |

| Spinner/prompts | `github.com/charmbracelet/huh` | Interactive forms and spinners |

### Relevant API Endpoints

All endpoints are at `https://api.tiddly.me` (configurable via `--api-url` or config).

**Authentication:**
- All endpoints accept `Authorization: Bearer <token>` (either Auth0 JWT or PAT)
- Token endpoints (`/tokens/*`) are Auth0-only -- PATs get 403
- Set `X-Request-Source: cli` header on all requests for audit trail

**Content (bookmarks/notes/prompts share the same pattern):**
- `GET /bookmarks/` -- List/search. Params: `q`, `tags`, `tag_match`, `sort_by`, `sort_order`, `offset`, `limit`, `view` (active/archived/deleted). Response: `{ items, total, offset, limit, has_more }`
- `GET /bookmarks/{id}` -- Full item with content. Supports `start_line`/`end_line` for partial reads
- `POST /bookmarks/` -- Create. Body: `{ url, title, description, content, tags }`. Returns 201
- `PATCH /bookmarks/{id}` -- Update fields. Supports `expected_updated_at` for optimistic locking
- `DELETE /bookmarks/{id}` -- Soft delete. `?permanent=true` for hard delete
- `POST /bookmarks/{id}/restore` -- Restore from trash
- `POST /bookmarks/{id}/archive` / `POST /bookmarks/{id}/unarchive`

**Unified search:** `GET /content/` -- Same params as individual list endpoints but searches across all types. Response items have `type` field (bookmark/note/prompt)

**Prompts (additional):**
- `GET /prompts/name/{name}` -- Get by name (primary lookup for prompts)
- `POST /prompts/{id}/render` -- Render Jinja2 template with arguments
- `GET /prompts/export/skills?client={client}&tags={tag}` -- Export as skills tar.gz/zip. Client values: `claude-code`, `claude-desktop`, `codex`

**Tokens (Auth0-only, all return 403 for PATs):**
- `POST /tokens/` -- Create PAT. Body: `{ name, expires_in_days }`. Response includes plaintext `token` (shown once)
- `GET /tokens/` -- List PATs (metadata only: id, name, token_prefix, last_used_at, expires_at)
- `DELETE /tokens/{id}` -- Revoke PAT

**Tags:**
- `GET /tags/` -- List all tags with counts. Optional: `content_types=bookmark`
- `PATCH /tags/{name}` -- Rename tag
- `DELETE /tags/{name}` -- Delete tag

**History:**
- `GET /history/` -- All user history. Params: `content_type`, `action`, `source`, date range, pagination
- `GET /history/{content_type}/{content_id}` -- Entity history
- `GET /history/{content_type}/{content_id}/version/{v}` -- Reconstruct content at version
- `POST /history/{content_type}/{content_id}/restore/{v}` -- Restore to version

**Other:**
- `GET /users/me` -- Current user info
- `GET /users/me/limits` -- Tier limits
- `GET /health` -- Health check
- `GET /consent/status` -- Consent check
- `POST /consent/me` -- Record consent

**Error codes:** 401 (invalid auth), 403 (PAT on Auth0-only endpoint), 409 (duplicate/conflict), 429 (rate limited with `Retry-After` header), 451 (consent required)

**Claude Code paths:**
- Global: `~/.claude/skills/{name}/SKILL.md`
- Project: `<project-root>/.claude/skills/{name}/SKILL.md`
- Legacy (still works, lower priority): `~/.claude/commands/{name}.md`

**Codex paths:**
- Global: `~/.codex/skills/{name}/SKILL.md`
- Project: `.agents/skills/{name}/SKILL.md` (note: `.agents/`, not `.codex/`)

**Claude Desktop:** ZIP file (manual upload — CLI provides instructions)

Client type values must exactly match: `claude-code`, `claude-desktop`, `codex` (from `backend/src/services/skill_converter.py` line 11).

### Implementation Outline

**1. Go module init and dependencies**

```bash
cd cli && go mod init github.com/shane-kercheval/tiddly/cli
```

Install: cobra, viper, go-keyring, golang.org/x/oauth2, lipgloss, huh.

**2. `main.go` + `cmd/root.go`**

Standard Cobra pattern. Root command registers global persistent flags:
- `--token` (string) -- override auth token
- `--api-url` (string) -- API base URL (default: `https://api.tiddly.me`)
- `--format` (string) -- output format: text, json (default: text)
- `--no-color` (bool) -- disable colored output
- `--verbose` (bool) -- verbose output

Root command `PersistentPreRunE` should initialize config (Viper) and set up the API client with resolved auth token.

**3. `internal/config/config.go`**

Config file location: `~/.config/tiddly/config.yaml` (respects `$XDG_CONFIG_HOME`). Store non-secret settings only:

```yaml
api_url: https://api.tiddly.me
format: text
```

Use Viper to read/write. Create config dir on first use if it doesn't exist.

**Config precedence order** (highest to lowest):
1. CLI flags (`--api-url`, `--format`, etc.)
2. Environment variables (`TIDDLY_API_URL`, `TIDDLY_FORMAT`)
3. Config file (`~/.config/tiddly/config.yaml`)
4. Defaults

Configure Viper binding to match this order. This applies to all non-auth settings (`api_url`, `format`, `editor`).

**4. `internal/auth/keyring.go`**

Wrap go-keyring for credential storage. Service name: `tiddly-cli`. Account names:
- `oauth-access-token` -- Auth0 JWT
- `oauth-refresh-token` -- Refresh token
- `pat` -- Stored PAT

File fallback for headless Linux: `~/.config/tiddly/credentials` (JSON, mode 0600). Keyring availability detection:
1. On Linux, check `DISPLAY` and `WAYLAND_DISPLAY` env vars -- if both unset, skip keyring entirely (no desktop session means no keyring service)
2. Wrap all keyring operations with a 3-second timeout as a safety net (go-keyring can hang on D-Bus rather than returning an error on some Linux configurations). Since go-keyring has no context parameter, implement via goroutine+select: run keyring operation in a goroutine, select on result channel vs `time.After(3 * time.Second)`
3. On timeout or error, fall back to file storage
4. Support hidden `--keyring` flag (`force`/`auto`/`file`) for users who need explicit control

**5. `internal/auth/device_flow.go`**

Implement Auth0 device authorization flow using `golang.org/x/oauth2`:

1. POST to `https://{auth0_domain}/oauth/device/code` with `client_id`, `scope=openid profile email offline_access`, `audience={auth0_audience}`
2. Display verification URL and user code
3. Attempt to open browser automatically (`open` on macOS, `xdg-open` on Linux, `rundll32` on Windows)
4. Poll `https://{auth0_domain}/oauth/token` every `interval` seconds with the `device_code`
5. Handle `authorization_pending`, `slow_down`, `expired_token`, `access_denied` responses
6. Polling uses a `context.Context` passed from the command, cancelled on SIGINT. Ctrl+C during polling prints "Login cancelled." and exits cleanly
7. On success, store access token and refresh token in keyring

Auth0 configuration values (domain, client ID, audience) should be hardcoded constants since this is a first-party CLI. These are not secrets. For dev/staging testing, allow hidden env var overrides: `TIDDLY_AUTH0_DOMAIN`, `TIDDLY_AUTH0_CLIENT_ID`, `TIDDLY_AUTH0_AUDIENCE` (undocumented, not exposed as flags).

**6. `internal/auth/token_manager.go`**

Token resolution function that returns the active token:

```go
func ResolveToken(flagToken string, preferOAuth bool) (token string, authType string, err error)
```

Default resolution order (`preferOAuth=false`):
1. `flagToken` if non-empty (auth type: "flag")
2. `TIDDLY_TOKEN` env var (auth type: "env")
3. Stored PAT from keyring (auth type: "pat")
4. Stored OAuth access token from keyring (auth type: "oauth") -- check expiry, attempt refresh if expired

OAuth-preferred resolution order (`preferOAuth=true`):
1. `flagToken` if non-empty (auth type: "flag")
2. `TIDDLY_TOKEN` env var (auth type: "env")
3. Stored OAuth access token from keyring (auth type: "oauth") -- check expiry, attempt refresh if expired
4. Stored PAT from keyring (auth type: "pat")

Commands that call Auth0-only endpoints (`tiddly tokens *`, PAT creation in `tiddly mcp install`) use `preferOAuth=true`. This prevents 403 errors when both a PAT and OAuth token are stored, since the default order would always pick the PAT.

For OAuth refresh: POST to Auth0 token endpoint with `grant_type=refresh_token`. Store BOTH the new access token AND the new refresh token returned by Auth0 (with refresh token rotation enabled, Auth0 invalidates the old refresh token on each use — failing to store the new one breaks auth on the next refresh cycle). Known limitation: if two CLI processes refresh simultaneously, the second will fail because its refresh token was already rotated. Surface this as "Session expired. Run 'tiddly login' to re-authenticate." (file lock for concurrent refresh is a future improvement).

**7. `internal/api/client.go`**

Minimal HTTP client for now (expanded in later milestones):

```go
type Client struct {
    BaseURL    string
    Token      string
    AuthType   string
    HTTPClient *http.Client
}

func (c *Client) Do(method, path string, body interface{}, result interface{}) error
```

Set headers on every request:
- `Authorization: Bearer {token}`
- `X-Request-Source: cli`
- `Content-Type: application/json`

Handle common error codes:
- 401 → "Session expired. Run 'tiddly login' to re-authenticate."
- 402 → Parse structured response: `"Quota exceeded: {resource} ({current}/{limit}). Upgrade at https://tiddly.me/pricing"`
- 403 → "This action requires browser login. Run 'tiddly login' (without --token)."
- 429 → Auto-retry up to 2 times using `Retry-After` header. Print "Rate limited, retrying in {n}s..." to stderr (not stdout, to avoid corrupting piped output). After max retries, show: "Rate limited. Try again in {n} seconds."
- 451 → "Please accept Terms of Service at https://tiddly.me/terms"

**8. `internal/api/users.go`**

Just `GET /users/me` and `GET /health` for now -- used by `tiddly auth status` and `tiddly status`.

**9. `cmd/login.go`**

```
tiddly login              # OAuth device flow
tiddly login --token X    # Direct PAT storage
```

For `--token`: trim leading/trailing whitespace (common copy-paste artifact — if trimmed, print "Trimmed whitespace from token."). Reject tokens with embedded spaces or newlines. Validate format (must start with `bm_`), call `GET /users/me` to verify it works, store in keyring.

For OAuth: run device flow, store tokens in keyring, call `GET /users/me` to display username.

**451 handling after login:** If `GET /users/me` returns 451 after successful authentication, parse the structured response and display the consent URL. Don't treat it as a login failure -- credentials are stored -- but warn that API calls will fail until consent is given:
```
✓ Logged in successfully.
⚠ Terms of Service acceptance required.
  Visit https://tiddly.me/terms to accept, then retry your command.
```

**10. `cmd/logout.go`**

Clear all keyring entries (oauth-access-token, oauth-refresh-token, pat).

**11. `cmd/auth.go`**

`tiddly auth status`:
- Show auth method (OAuth/PAT/none)
- Show user email (from `GET /users/me`)
- Show token expiry if OAuth
- Show API URL

The real implementations wrap the stdlib (`os/exec`, `golang.org/x/term`, `go-keyring`, etc.). Tests inject mocks. Use constructor injection — structs accept interfaces in their constructor, not via global state.

**2. Mock API server helper (`internal/testutil/mock_api.go`)**

A builder that creates an `httptest.NewServer` with route-based response matching:

```go
func NewMockAPI(t *testing.T) *MockAPI
func (m *MockAPI) On(method, path string) *RouteBuilder
func (r *RouteBuilder) RespondJSON(status int, body interface{}) *RouteBuilder
func (r *RouteBuilder) RespondError(status int, message string) *RouteBuilder
func (r *RouteBuilder) AssertHeader(key, value string) *RouteBuilder
func (r *RouteBuilder) AssertCalled(times int)          // verify in t.Cleanup
func (m *MockAPI) URL() string                          // base URL for api.NewClient
```

Usage in tests:
```go
mock := testutil.NewMockAPI(t)
mock.On("GET", "/users/me").
    AssertHeader("Authorization", "Bearer test-token").
    RespondJSON(200, map[string]any{"email": "user@example.com"})

client := api.NewClient(mock.URL(), "test-token", "cli")
user, err := client.GetMe()
```

Every `internal/api/*_test.go` and `cmd/*_test.go` uses this. No test builds its own `httptest.NewServer` from scratch.

**3. Command test helper (`internal/testutil/cmd_helper.go`)**

Execute Cobra commands in-process (NOT by shelling out to the compiled binary). Capture stdout/stderr. Inject mock dependencies:

```go
type CmdResult struct {
    Stdout   string
    Stderr   string
    ExitCode int
}

func RunCmd(t *testing.T, opts CmdOpts, args ...string) CmdResult

type CmdOpts struct {
    MockAPI    *MockAPI          // nil = no API (for --help tests)
    Creds      *MockCredStore    // pre-populated credentials
    Env        map[string]string // env var overrides
    Stdin      io.Reader         // for interactive input testing
    IsTTY      bool              // controls TTY detection mock
}
```

Usage:
```go
result := testutil.RunCmd(t, testutil.CmdOpts{
    MockAPI: mock,
    Creds:   testutil.CredsWithPAT("bm_test123"),
    IsTTY:   false,

assert.Equal(t, 0, result.ExitCode)
assert.Contains(t, result.Stdout, `"title"`)
```
